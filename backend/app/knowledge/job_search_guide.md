# AI 求职知识库：基础求职指南

## 一、没有实习经历时如何写简历

如果求职者没有正式实习经历，可以重点突出个人项目、课程项目、毕业设计、开源练习项目和可展示作品。

简历中不要直接写“没有实习经历”，而是应该通过项目经历证明自己的动手能力。

项目描述要包含：
1. 项目背景
2. 使用技术
3. 个人负责内容
4. 解决的问题
5. 最终成果
6. 可以量化的数据

例如：
- 完成前后端联调
- 接入大模型 API
- 实现多轮对话
- 设计用户画像结构
- 完成项目部署
- 使用 Git 管理代码

## 二、AI 全栈项目如何写进简历

AI 全栈项目不能只写“调用了大模型 API”，否则含金量会显得不够。

更好的写法应该突出：
1. 前端页面设计
2. 后端接口设计
3. 大模型 API 接入
4. Prompt 工程
5. 多轮上下文管理
6. RAG 知识库检索
7. 用户画像
8. 数据库存储
9. Docker 部署
10. 项目工程化结构

可以这样描述：

基于 Next.js、FastAPI 和大模型 API 开发 AI 求职智能体平台，实现用户画像管理、岗位 JD 分析、简历项目优化、面试问答准备和 RAG 知识库问答等功能。后端采用模块化结构封装模型调用、Prompt 构造和知识库检索逻辑，前端实现多轮聊天、岗位输入、Markdown 渲染和本地状态保存。

## 三、岗位 JD 分析应该看什么

分析一个岗位 JD 时，应该重点关注：

1. 岗位名称
2. 核心职责
3. 必备技术栈
4. 加分项
5. 项目经验要求
6. 学历和经验要求
7. 业务方向
8. 隐含能力要求

例如，AI 应用开发岗位通常关注：
- Python 后端能力
- API 接口开发能力
- 大模型 API 接入经验
- Prompt 设计能力
- RAG 或 Agent 相关理解
- 前后端联调能力
- 项目落地能力

## 四、面试介绍 AI 项目的方法

介绍 AI 项目时，不要只说“我做了一个聊天机器人”。

应该按照下面结构讲：

1. 项目背景：为什么做这个项目
2. 用户痛点：解决什么问题
3. 技术架构：前端、后端、模型、数据库、部署
4. 核心功能：具体实现了哪些功能
5. 个人工作：自己负责了什么
6. 技术难点：遇到了什么问题
7. 解决方案：如何解决
8. 项目成果：最终效果是什么
9. 后续优化：还能怎么继续完善

## 五、没有实习经历怎么回答

如果面试官问“你没有实习经历，怎么证明你能胜任工作”，可以这样回答：

虽然我目前还没有正式实习经历，但我一直在通过完整项目训练自己的工程能力。比如我独立开发了 AI Career Agent 项目，从前端页面、后端接口、大模型 API 接入、Prompt 设计、多轮对话、用户画像、岗位匹配分析到 RAG 知识库问答，都是自己一步一步实现的。

这个项目让我理解了真实 AI 应用从用户输入到模型调用再到结果展示的完整链路，也训练了我独立排查问题、阅读文档和持续迭代项目的能力。
